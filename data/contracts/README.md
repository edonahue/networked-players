# Data contracts

Schemas will be added here when the first collection-slice vertical path is implemented.

Initial contracts should cover:

- stable source and normalized identifiers;
- releases and release versions;
- contributors and credited names;
- normalized roles without discarding source text;
- credit evidence and provenance;
- graph snapshot manifests;
- static challenge and answer artifacts.

Do not design a comprehensive schema before testing it against a small, manually verified slice.
